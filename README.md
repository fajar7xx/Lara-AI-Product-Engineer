# AI Product Engineer

Turn raw product ideas into structured documentation and landing page prototypes using AI.

AI Product Engineer is a Laravel-based tool that transforms unstructured transcripts or product descriptions into polished deliverables — **Product Requirements Documents (PRD)**, **User Stories**, **Functional Requirements**, and **HTML landing pages** — all generated by AI and refinable through an interactive clarification flow.

## Features

- **AI-Powered Extraction** — Paste any product transcript; AI extracts project name, summary, goals, features, and target users.
- **Clarification Flow** — Review and refine AI-extracted context before generation. Choose template families (landing, app shell) and design systems (minimal, modern, corporate).
- **Multi-Output Generation** — Generate structured documents on demand:
  - Product Requirements Document (PRD)
  - User Stories
  - Functional Requirements
  - HTML Landing Pages
- **HTML Page Preview** — Preview generated landing pages directly in the browser with live rendering.
- **PDF Export** — Export generated outputs as PDF documents.
- **Full Authentication** — Powered by Laravel Fortify with email verification, password reset, two-factor authentication (2FA/TOTP), and passkeys (WebAuthn).
- **Profile & Appearance Settings** — Manage profile info, security settings, and UI theme preferences.
- **Revision Tracking** — Each generation increments revision number; clarification snapshots are preserved for reproducibility.
- **AI Provider Failover** — Primary: OpenRouter (DeepSeek). Fallback: Gemini. Automatic retry on transient provider failures.

## Getting Started

### Installation

```bash
# Clone the repository
git clone <repository-url>
cd ai-product-engineer

# Install PHP dependencies
composer install

# Set up environment
cp .env.example .env
php artisan key:generate

# Install Node dependencies
npm install

# Build frontend assets
npm run build

# Run migrations
php artisan migrate

# (Optional) Seed database
php artisan db:seed
```

### Configuration

Configure your AI provider(s) in `.env`:

```env
# OpenRouter (primary)
AI_PROVIDER_OPENROUTER_KEY=your-openrouter-key

# Gemini (fallback)
AI_PROVIDER_GEMINI_KEY=your-gemini-key

# Database
DB_CONNECTION=pgsql
DB_HOST=127.0.0.1
DB_PORT=5432
DB_DATABASE=ai_product_engineer
DB_USERNAME=postgres
DB_PASSWORD=
```

### Development Server

Run all services simultaneously:

```bash
composer run dev
```

This starts the Artisan server, queue listener, log watcher (Pail), and Vite dev server concurrently.

### Running Tests

```bash

composer run test

php artisan test --compact --filter=test_name
```

## Architecture

### Application Flow

1. **Transcript Input** — User pastes a raw product idea/transcript.
2. **AI Extraction** — `TranscriptExtractionService` sends the transcript to an AI agent which extracts structured context (project name, summary, goals, features, target users, template family + design system recommendations).
3. **Clarification** — User reviews and refines the extracted context, chooses template family and design system.
4. **Generation** — `SessionGenerationService` dispatches AI agents in parallel (or sequentially per type) to generate each output:
   - `PrdGenerationAgent` → PRD markdown
   - `UserStoriesGenerationAgent` → User Stories markdown
   - `FunctionalRequirementsGenerationAgent` → Functional Requirements markdown
   - `HtmlPageContentAgent` → structured page data → assembled by `HtmlAssemblyService` into a Blade-rendered HTML page
5. **Preview & Export** — Users preview generated HTML pages or export outputs as PDF.

### AI Agents

The application uses the Laravel AI SDK with structured output agents:

- `TranscriptExtractionAgent` — Extracts structured context from raw transcripts.
- `PrdGenerationAgent` — Generates a Product Requirements Document.
- `UserStoriesGenerationAgent` — Generates user stories.
- `FunctionalRequirementsGenerationAgent` — Generates functional requirements.
- `HtmlPageContentAgent` — Generates structured HTML page content.

Each agent uses a provider chain with failover support (OpenRouter → Gemini).

### Status Lifecycle

**TranscriptSession**: `draft` → `extracting` → `clarifying` → `generating` → `completed` / `partial` / `failed`

**GenerationOutput**: `pending` → `processing` → `completed` / `failed`
